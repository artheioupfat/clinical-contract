L'éditeur est une page statique déployable sur GitHub Pages. Les contrats et fichiers sont traités localement dans votre navigateur, sans serveur applicatif ni collecte de données.

Workflow

Définissez le fichier attendu. Vérifiez le fichier reçu.

clinical-contract transforme une attente métier en contrôle reproductible, lisible par les équipes projet, data et cliniques.

01

Rédiger le contrat

Décrivez la table attendue, les colonnes, les types et les règles qualité dans un YAML versionnable.

02

Charger les données

Importez un fichier CSV ou Parquet et prévisualisez rapidement sa structure avant le contrôle.

03

Lancer les contrôles

Comparez le schéma et exécutez les règles qualité SQL pour produire un résultat exploitable.

Private by design

Un éditeur statique, sans serveur applicatif.

La page peut être déployée sur GitHub Pages. Les contrats et fichiers restent dans le navigateur : PyScript, Pyodide et DuckDB exécutent les validations localement, sans envoyer les données vers un backend.

Fondation

Un contrat lisible avant l'échange, un contrôle reproductible après réception.

clinical-contract s'appuie sur les concepts du standard Data Contract pour formaliser ce qui est attendu dans un fichier de données cliniques.

Standard

Structurer le besoin dans un format YAML compréhensible et versionnable.

Contrat

Décrire les colonnes, les types, les descriptions et les règles de qualité attendues.

Vérification

Comparer le contrat au fichier reçu et produire un résultat lisible par les équipes.

Python library + CLI

Le même contrat dans l'éditeur, dans Python et dans votre terminal.

clinical-contract existe aussi comme bibliothèque Python et comme commande CLI. L'objectif est simple : rédiger un contrat une fois, puis l'utiliser partout pour valider sa structure et vérifier un fichier CSV ou Parquet.

validate 
uv tool install --python python3.11 clinical-contract
