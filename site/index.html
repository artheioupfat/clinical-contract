Éditeur Lire la documentation

Une page statique pour rédiger un contrat, valider sa structure et vérifier un fichier CSV ou Parquet localement dans le navigateur.

Fondation

Un contrat lisible avant l'échange, un contrôle reproductible après réception.

clinical-contract s'appuie sur les concepts du standard Data Contract pour formaliser ce qui est attendu dans un fichier de données cliniques.

Standard

Structurer le besoin dans un format YAML compréhensible et versionnable.

Contrat

Décrire les colonnes, les types, les descriptions et les règles de qualité attendues.

Vérification

Comparer le contrat au fichier reçu et produire un résultat lisible par les équipes.

Python library + CLI

Le même contrat dans l'éditeur, dans Python et dans votre terminal.

clinical-contract existe aussi comme bibliothèque Python et comme commande CLI. L'objectif est simple : rédiger un contrat une fois, puis l'utiliser partout pour valider sa structure et vérifier un fichier CSV ou Parquet.

validate 
uv tool install --python python3.11 clinical-contract
